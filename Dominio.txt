Verifiqué la configuración de Apache:

Ejecuté "sudo apachectl configtest" para comprobar la sintaxis.

Recargué Apache con "sudo systemctl reload apache2".

Intenté resolver problemas DNS:

Ejecuté "sudo resolvectl flush-caches" para limpiar la caché DNS.

Usé el comando "dig vytalgym.es" para verificar la resolución del dominio.

Verifiqué el estado del firewall:

Ejecuté "sudo ufw status" y vi que estaba inactivo.

Edité la configuración de Apache:

Usé "sudo nano /etc/apache2/sites-available/vytalgym.es.conf" para editar el archivo de configuración del sitio virtual.

Usé "sudo tail -f /var/log/apache2/error.log" para ver los últimos mensajes de error.

Ajusté los permisos de los archivos del proyecto:

Ejecuté "sudo chown -R www-data:www-data /var/www/vytalgym" para cambiar el propietario.

Usé "sudo chmod -R 755 /var/www/vytalgym" para ajustar los permisos.

Edité el archivo WSGI de la aplicación:

Usé "sudo nano /var/www/vytalgym/Programacion/py/app.wsgi" para editar el archivo.

Instalé dependencias de Python:

Activé el entorno virtual con "source .venv/bin/activate".

Ejecuté "pip install -r requirements.txt" para instalar las dependencias.

Probé la aplicación Flask en modo de desarrollo:

Ejecuté "python app.py" para iniciar el servidor de desarrollo en http://127.0.0.1:8000.

Configuré el firewall para permitir tráfico HTTP:

Ejecuté "sudo ufw allow 80/tcp" para abrir el puerto 80.

Ahora se puede ver el servidor de desarollo en vytalgym.es
